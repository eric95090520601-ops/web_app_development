# 產品需求文件 (PRD) - 食譜收藏檢索系統

## 1. 專案概述

### 1.1 背景與動機
現代人生活忙碌，有時雖然想親自下廚，卻不知道如何利用現有食材，或者在網路上看到喜歡的食譜後難以有系統地集中管理與搜尋。為了幫助熱愛烹飪或想開始嘗試煮菜的使用者更方便地尋找靈感、管理心儀食譜並與他人交流，我們決定開發一個「食譜收藏檢索系統」。

### 1.2 目標用戶
- 平時有在煮菜，需要靈感或想分享自己食譜的人。
- 想嘗試煮菜的新手，需要透過簡單的菜名或手邊現有食材來尋找教學的人。

### 1.3 核心價值主張
提供一個簡單易用的平台，讓使用者能依據手邊的「食材」或心中的「菜名」快速找到對應的食譜，同時能方便地「收藏」他人的心血結晶，並「分享」自己的獨門料理與心得（留言互動），打造一個充滿溫度的烹飪交流社群。

---

## 2. 功能需求

### 2.1 主要功能清單
1. **食材檢索**：依據食材來查找相關食譜。
2. **菜名檢索**：依據菜名來查找相關食譜。
3. **食譜上傳**：使用者能夠建立並上傳自己的食譜分享給大家。
4. **食譜收藏**：使用者能夠儲存/收藏他人的食譜至個人帳號中。
5. **留言互動**：使用者能夠在他人的食譜底下留言，進行討論或給予回饋。
6. **會員系統**：使用者註冊、登入及個人頁面（用來管理自己的上傳與收藏）。

### 2.2 使用者故事 (User Stories)
- **食材檢索**：作為一個 [想清冰箱的使用者]，我希望 [能輸入多個食材名稱進行搜尋]，以便 [找到能消耗這些食材的食譜]。
- **菜名檢索**：作為一個 [想吃特定料理的使用者]，我希望 [能直接搜尋菜色名稱（如：番茄炒蛋）]，以便 [快速找到做法]。
- **食譜上傳**：作為一個 [熱愛分享的料理好手]，我希望 [能填寫圖文並茂的食譜（包含食材、步驟）並發布]，以便 [讓其他人也能照著做]。
- **食譜收藏**：作為一個 [尋找靈感的使用者]，我希望 [能一鍵收藏喜歡的食譜]，以便 [未來要煮飯時能快速找出來參考]。
- **留言互動**：作為一個 [看完食譜的使用者]，我希望 [能在食譜下方留言分享試做心得或發問]，以便 [與作者或其他使用者交流]。
- **會員系統**：作為一個 [平台使用者]，我希望 [能有自己的帳號]，以便 [管理我上傳的食譜與收藏的清單]。

## 3. 非功能需求

### 3.1 技術限制
- **前端技術**：HTML, CSS (可選用 Bootstrap/Tailwind 等簡單框架), JavaScript
- **後端框架**：Flask (Python)
- **模板引擎**：Jinja2
- **資料庫**：SQLite (適合初期輕量化、快速開發與部署)

### 3.2 效能與安全考量
- **圖片處理**：若允許使用者上傳食譜圖片，需限制檔案大小與格式（如 JPG, PNG，最大 5MB），以節省儲存空間與頻寬。
- **防止 SQL Injection**：使用 ORM（如 SQLAlchemy）或參數化查詢來與 SQLite 互動，確保資料庫安全。
- **密碼安全**：使用者密碼需經過 Hash（如 bcrypt 或 werkzeug.security）後再存入資料庫，絕不儲存明碼。
- **XSS 防護**：使用 Jinja2 預設的自動跳脫（autoescape）機制處理使用者輸入的留言與食譜內容，避免跨站腳本攻擊。

## 4. MVP 範圍 (Minimum Viable Product)

### 4.1 Must Have (必須具備 - 核心價值)
- 使用者註冊與登入功能
- 依據菜名或關鍵字搜尋食譜
- 食譜的 CRUD（建立、讀取、更新、刪除自己上傳的食譜）
- 瀏覽他人公開的食譜

### 4.2 Should Have (應該具備 - 完整體驗)
- 依據「食材」進行標籤化或進階搜尋
- 收藏/取消收藏他人食譜的功能，並能在個人頁面檢視「我的收藏」
- 在食譜下方留言的功能

### 4.3 Nice to Have (有了更好 - 進階功能)
- 食譜評分系統（例如：給 1~5 顆星）
- 依據熱門程度或最新發布來排序食譜
- 忘記密碼/密碼重置功能
- 食譜的份量換算功能（例如輸入 2 人份自動調整食材克數）
